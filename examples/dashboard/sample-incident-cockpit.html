SafeOps Incident Cockpit Sample

Incident Summary

Incident: inc_checkout-api_demo

Service: checkout-api

Root cause: REDIS_URL was removed or misconfigured after deployment.

Verification healthy

CI/CD Correlation

Required environment variable REDIS_URL exists in the known-good manifest but is missing from the deployed/broken manifest.

Approved Remediation

kubectl -n demo set env deployment/checkout-api REDIS_URL=redis://redis.demo.svc.cluster.local:6379
