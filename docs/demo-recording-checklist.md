# Demo Recording Checklist

## Before recording

- Confirm `git status` is clean.
- Confirm GitHub Actions is green.
- Run the investor demo once.
- Open the redesigned Incident Cockpit.
- Close unrelated browser tabs.
- Hide tokens, private files, and personal notifications.
- Use a large readable terminal font.

## Screen order

1. GitHub README.
2. GitHub Actions green check.
3. Terminal running `./scripts/demo_run_investor.sh`.
4. Incident Cockpit dashboard.
5. Prevention PR output/package.

## Key talking points

- SafeOps is not uncontrolled AI automation.
- SafeOps is evidence-driven.
- SafeOps requires approval.
- SafeOps checks policy before execution.
- SafeOps verifies recovery.
- SafeOps creates an audit trail.
- SafeOps generates prevention artifacts.

## After recording

- Watch the video once end-to-end.
- Confirm audio is clear.
- Confirm terminal text is readable.
- Confirm no token or private data is visible.
- Save a short public link for reviewers.
